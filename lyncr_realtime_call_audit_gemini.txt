# Lyncr Real-Time Call Architecture — Operational Audit

Scope: Call Quality Tracking + Call Flow Control across Platform Admin, Business Owner, Receptionist, and Field Technician consoles.
Production mode: Telnyx-first; tenants on Call Control when ZING_INBOUND_CALL_CONTROL=1.

## Executive Summary

| Layer | Status |
|-------|--------|
| Inbound routing | Two parallel pipelines — TeXML (full feature set) and Call Control (simpler, PSTN-only dial) |
| Call state in DB | call_logs is the shared source of truth (ringing → answered_at → ended_at) |
| Real-time layer | Pusher Channels on owner / receptionist / technician channels; Platform Admin has NO Pusher |
| Call quality | Setup latency KPIs only (setup_duration_ms, answer rate); NO RTP/MOS/jitter, NO persisted hangup_cause |
| Cross-console sync | Strong at disposition handoff (ai_leads); weak at live-call lifecycle (receptionist end events, WebRTC intake, tech job updates) |

### 1.2 Inbound SIP Invite → DID Resolution → Active State

The app never parses raw SIP INVITEs. Telnyx terminates SIP/PSTN and invokes webhooks.

FLOW (TeXML default):
  Caller PSTN → Telnyx DID
    → GET /api/voice/telnyx/greet (Edge, instant answer + Redirect)
    → POST/GET /api/voice/telnyx/incoming
         → resolveCalledParty() + getIncomingRoutingForVoiceWebhook(DID)
         → insertCallLog(status=ringing, first_ring_at=now)
         → optional branded greeting pass
         → Dial receptionist CELL (PSTN) or WEB (SIP WebRTC) or Owner
         → on no-answer: Dial action → /api/voice/telnyx/fallback/u/{userId}/...
    Parallel: Telnyx status_callback → POST /api/voice/telnyx/status
      → recordCallStatusEvent (answered_at, ended_at, setup_duration_ms)
      → updateCallLog + Pusher call-completed + SMS disposition

FLOW (Call Control, ZING_INBOUND_CALL_CONTROL=1):
  Caller PSTN → Telnyx DID → Call Control app
    → POST /api/webhooks/telnyx/voice (JSON Voice API v2)
  call.initiated → getIncomingRoutingForVoiceWebhook(to) → insertCallLog(ringing) → REST answer
  call.answered → optional speak(greeting) OR dialTechnicianLeg(PSTN)
  call.speak.ended → dial outbound leg (link_to inbound, bridge_on_answer)
  call.bridged → persistCallControlBridged → recordCallStatusEvent(answered)
  call.hangup → finalizeCallControlCallLog → map hangup_cause → status, duration

Call Control client_state phases (lib/telnyx-call-control-state.ts):
  await_caller_answered → await_greeting_end → await_dial_end → (bridge | voicemail) → recording → hangup

DID metadata resolution — getIncomingRoutingForVoiceWebhook() in lib/db.ts:
  1. Short-TTL in-memory cache
  2. Fast path: denormalized snapshot on phone_numbers (inbound_dial_e164, receptionist id, fallback, ring timeout) — scripts 036, 050, 081
  3. Full join: phone_numbers + users + routing_config + receptionists + organizations + onboarding_profiles
  4. Fallback: onboarding_profiles.reserved_number
  Matching uses E.164 normalization + digit-key loose match (last 10 digits).

Active call state columns (call_logs, extended by scripts/007-call-quality-metrics.sql):

| Phase | DB signal | Set by |
|-------|-----------|--------|
| Ringing | status = ringing, first_ring_at | insertCallLog on first webhook |
| Connected | answered_at, status answered/in-progress | recordCallStatusEvent or Call Control call.bridged |
| Live (open leg) | ended_at IS NULL | Until terminal webhook |
| Completed | ended_at, terminal status, duration_seconds | Status webhook / Call Control call.hangup / TeXML fallback |

Auxiliary state:
  telnyx_inbound_dial_caller_done — prevents TeXML re-dial loops (scripts/018)
  Realtime (not DB): Pusher via lib/call-telemetry-realtime.ts and app/actions/call-events.ts

WebRTC receptionist path (TeXML only today):
  receptionists.routing_endpoint = WEB + sip_username from lib/telnyx-sip-provisioning.ts
  Incoming route builds sip:{username}@{domain} via resolveReceptionistSipUri() → Dial Sip TeXML
  Browser: lib/webrtc/use-telnyx-webrtc.ts (client-side; errors stay in UI, not DB)
  Call Control path: dials PSTN E.164 only — no SIP/WebRTC dial in Call Control inbound today.

### 1.3 Call Quality Metrics — Capture & Logging

What IS captured today:

| Metric | Where captured | DB column | Notes |
|--------|----------------|-----------|-------|
| Talk duration | Status webhook, TeXML Dial action, Call Control hangup | duration_seconds | GREATEST merge on updates |
| Setup latency | recordCallStatusEvent on answer | setup_duration_ms | answered_at − first_ring_at |
| Post-dial delay | Same function | post_dial_delay_ms | CURRENTLY DUPLICATES setup formula — not true bridge delay |
| Answer rate / p95 setup | Aggregated query | — | GET /api/voice/quality → getCallQualitySummary() |
| Recording length | Recording webhook | recording_duration_seconds | Voicemail path |
| Hangup cause | Call Control parse only | NOT STORED | Mapped to coarse status in telnyx-call-control-call-log.ts |
| RTP quality (MOS, jitter, packet loss) | TeXML dial attributes only | NOT measured/stored | lib/telnyx-inbound-media-quality.ts configures media, does not report |
| Connection/routing errors | Server logs | NOT persisted | console.error / structured JSON only |

Quality API: app/api/voice/quality/route.ts — owner-authenticated aggregates over call_logs (requires migration 007).

Core timing writer: recordCallStatusEvent() in lib/db.ts (~L3319):
  Sets status, duration_seconds, answered_at, ended_at, setup_duration_ms, post_dial_delay_ms from first_ring_at.

Hangup cause mapping (NOT persisted as raw value) — lib/telnyx-call-control-call-log.ts:
  normal_clearing → completed
  user_busy → busy
  no_answer / timeout / time_limit → no-answer
  call_rejected → failed
  originator_cancel → completed if had conversation else canceled

## 3. Gap Analysis & JSONB Consistency

### 3.1 Structural Data Silos

| Gap | Consoles affected | Risk |
|-----|-------------------|------|
| Dual inbound pipelines (TeXML vs Call Control) | All | Call Control lacks WebRTC SIP dial, routing pool, full AI fallback — behavior diverges by env flag |
| Admin = DB poll, Owner = Pusher aggregates | Admin vs Owner | Admin sees live list; owner sees counts only |
| handleCallEnded never wired | Receptionist | Intake UI may stick after hangup |
| WebRTC missing call-connected | Receptionist (WEB) | Full intake form may never open |
| job-updated never published | Field Tech | Status changes lag up to 20s |
| Mid-call intake silent to tech | Receptionist → Tech | No partial dispatch until log-job |
| hangup_cause not in DB | Admin, Owner analytics | Cannot diagnose SIP failures |
| post_dial_delay_ms = setup duplicate | Quality KPIs | Misleading answer speed metrics |
| TeXML fallback skips recordCallStatusEvent | Quality tracking | answered_at depends on status callback |
| Routing edits mid-call | Owner | Next call only — no in-call indicator |
| Stuck ringing rows (historical) | Admin Live Traffic | ended_at NULL rows appear live until backfilled |
| Mobile app no Pusher | Field Tech | Always poll-dependent |

## 4. Code Injection Recommendations

Prioritized bridges to unify the four consoles.

### P0 — Close Live-Call Lifecycle Gaps

1. Wire handleCallEnded on every terminal path
   Inject into:
     app/api/voice/telnyx/status/route.ts (terminal status)
     lib/telnyx-call-control-call-log.ts → finalizeCallControlCallLog
     lib/telnyx-fallback-dial-action.ts (TeXML fallback completion)
   Pattern: await handleCallEnded({ receptionistId, callLogId: row.id })

2. Fire handleCallConnected on WebRTC bridge
   Option A: Client POST /api/receptionist/call-connected after WebRTC active in use-telnyx-webrtc.ts
   Option B: Server-side on call.bridged when routed endpoint is WEB
   Server validates active call_logs row for receptionist → handleCallConnected

3. Publish job-updated to technician channel
   Inject into paths that publish owner job-status-updated:
     app/api/tech/jobs/[id]/route.ts
     app/api/owner/jobs/[id]/status/route.ts
     app/api/tech/location/route.ts (geofence auto-arrive)
   Pattern: await publishTechnicianEvent(techUserId, "job-updated", { leadId, status })

### P1 — Unify Call State Visibility

4. Shared call session broadcaster (new lib/call-session-broadcast.ts)
   Normalized payload on ring, answer, end:
     call_log_id, owner_user_id, receptionist_id, organization_id
     phase: ringing | connected | completed
     from_number, to_number, routed_to_name, duration_seconds
   Subscribe from Admin (new admin-network channel), Owner, Receptionist.

5. Admin Live Traffic → hybrid push + poll
   Keep 5s poll as fallback; add Pusher on admin-network channel fed by same broadcaster.

### P2 — Call Quality Tracking

6. Persist telephony diagnostics on call_logs (new migration e.g. scripts/082-call-telephony-diagnostics.sql):
   hangup_cause TEXT
   sip_response_code INTEGER
   provider_error_code TEXT
   routing_path TEXT  -- texml | call_control
   Write in telnyx-call-control-call-log.ts and app/api/voice/telnyx/status/route.ts

7. Fix post_dial_delay_ms semantics in recordCallStatusEvent:
   setup_duration_ms = answered_at − first_ring_at
   post_dial_delay_ms = bridged_at − dial_started_at (needs dial_started_at column or Call Control client_state timestamp)

8. Surface quality on Owner + Admin:
   Extend GET /api/voice/quality with hangup-cause breakdown
   Add GET /api/admin/call-quality for cross-tenant aggregates

### P3 — JSONB / Handoff Integrity

9. Bidirectional call ↔ lead link:
   ALTER TABLE call_logs ADD COLUMN ai_lead_id UUID REFERENCES ai_leads(id);
   Set on first intake save; enables Admin Live Traffic to show intake preview.

10. Single writer for disposition in lib/call-disposition.ts:
    One transaction: call_logs.disposition + ai_leads columns + collected JSONB
    Then fan-out Pusher to owner, receptionist, tech
    Replace scattered writes in log-job route.

11. Mid-call intake → owner preview event:
    On POST /api/receptionist/intake publish owner intake-draft-updated { callLogId, fields }

### P4 — Pipeline Convergence

12. Call Control feature parity before full rollout:
    Port from TeXML: WebRTC SIP dial, routing pool, press-1 screen, full fallback chain
    OR document Call Control as owner-cell-only and hide WEB receptionist toggle when flag is on.
